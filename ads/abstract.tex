\begin{abstract}
Diese Arbeit beschäftigt sich mit der Konstruktion und Evaluierung einer Hybrid Cloud-Lösung, bei der die Anwendung in einer öffentlichen Cloud bereitgestellt wird, der Speicher aber lokal in einem Rechenzentrum verbleibt.
Hierbei soll überprüft werden, auf welche Arten die Vorteile eines komplexen Clusters von Speichersystemen in der Cloud Verwendung finden können und auf welche Weisen sich die Anbindung umsetzen lässt. Es wird ein lokaler Speichercluster mit IBM Spectrum Scale aufgesetzt und eine Demoanwendung für die öffentlichen Cloud von IBM, Bluemix, entwickelt. Die Applikation demonstriert den Zugriff auf Testdaten mithilfe des IBM Cloud Object Storages.
Der beschriebene Aufbau wurde erfolgreich umgesetzt und eine Applikation erzeugt, die Bilddaten aus Scale oder vom Nutzer hochgeladen analysieren und betrachten kann. Für die Analyse wird ein Deep-Learning Watson Service verwendet, der mithilfe der REST-Schnittstelle der Node.js Anwendung angesprochen wird.
Es hat sich ergeben, dass ein Hybrid-Cloud Setup in dieser Form umsetzbar ist, auch wenn aufgrund der komplizierten Architektur Mehraufwand bei der Entwicklung entstehen kann.
\end{abstract}
